Como o Score Combinatório funciona

O dashboard combina três camadas de análise para rankear cada opção:

1. Probabilidade histórica (base: planilha ESTATISTICAS)

Cada gap de abertura (+1%, -2%, etc.) tem um histórico de ~12 anos de comportamento. O sistema localiza o bucket de gap atual e usa a probabilidade de fechamento positivo em D+1, D+2 e D+3. Quanto maior a probabilidade E maior o número de amostras, maior o peso neste componente. Peso: 35 pts máx.

2. Estrutura da opção (Black-Scholes)

Para cada opção candidata, calcula: prêmio, delta real em R$, gamma, theta, vega, e o payoff esperado se o ativo mover conforme o histórico. O delta ideal para compra seco está entre 0.25-0.45 (opção OTM com boa alavancagem). IV próxima ou abaixo da HV indica prêmio razoável. Peso: 35 pts máx.

3. Contexto de mercado

Horário (abertura 09:01-09:30 = peso extra), magnitude do gap vs. a distribuição histórica, e o retorno estimado na opção se o ativo se mover a favor nos próximos 1-3 dias. Peso: 30 pts máx.

Delta em R$ — o que significa

O Delta mostrado é em reais por real do ativo, igual ao Profit. Exemplo: delta 0.35 significa que a opção valoriza R$ 0,35 para cada R$ 1,00 de alta no ativo. Para 100 opções, a sensibilidade é R$ 35 por R$ 1 de movimento.

Dados ao vivo via Profit Pro

Para preços em tempo real, rode o profit_bridge.py na sua máquina com o Profit aberto. Ele cria um WebSocket local que o dashboard consulta. Para acesso público, use ngrok http 8765 e cole a URL no campo de configuração. Sem o bridge, o dashboard usa a API pública brapi.dev como fallback.

# Instalar dependências
pip install websockets requests

# Rodar o bridge
python profit_bridge.py

# Expor publicamente
ngrok http 8765
